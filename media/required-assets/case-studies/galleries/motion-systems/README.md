# Motion Systems Gallery Assets

This folder should contain the following gallery images for the Motion Systems case study.

## Required Files

1. **motion-library-hero.jpg**
   - Dimensions: 2400 × 1600px (landscape) or 2400 × 2400px (square)
   - Min: 1200 × 800px
   - Format: JPG (85-90% quality)
   - Description: Motion Systems Library hero visual

2. **caribbean-pools-drone-shots.jpg**
   - Dimensions: 2400 × 1600px (landscape)
   - Min: 1200 × 800px
   - Format: JPG (85-90% quality)
   - Description: Caribbean Pools drone cinematography showcase

3. **workflow-explainers.jpg**
   - Dimensions: 2400 × 1600px (landscape)
   - Min: 1200 × 800px
   - Format: JPG (85-90% quality)
   - Description: Workflow explainer motion graphics examples

4. **product-system-motion.jpg**
   - Dimensions: 2400 × 1600px (landscape)
   - Min: 1200 × 800px
   - Format: JPG (85-90% quality)
   - Description: Product and system motion graphics

5. **kids-ip-motion-tests.jpg**
   - Dimensions: 2400 × 1600px (landscape)
   - Min: 1200 × 800px
   - Format: JPG (85-90% quality)
   - Description: Kids' IP motion test animations

6. **ai-assisted-frames.jpg**
   - Dimensions: 2400 × 1600px (landscape)
   - Min: 1200 × 800px
   - Format: JPG (85-90% quality)
   - Description: AI-assisted animation frames and hybrid production work

## Upload Path

All files should be uploaded to R2 at:
```
portfolio/motion-systems/{filename}
```

## Upload Command

```bash
npm run r2:upload ./motion-library-hero.jpg portfolio/motion-systems/motion-library-hero.jpg
npm run r2:upload ./caribbean-pools-drone-shots.jpg portfolio/motion-systems/caribbean-pools-drone-shots.jpg
npm run r2:upload ./workflow-explainers.jpg portfolio/motion-systems/workflow-explainers.jpg
npm run r2:upload ./product-system-motion.jpg portfolio/motion-systems/product-system-motion.jpg
npm run r2:upload ./kids-ip-motion-tests.jpg portfolio/motion-systems/kids-ip-motion-tests.jpg
npm run r2:upload ./ai-assisted-frames.jpg portfolio/motion-systems/ai-assisted-frames.jpg
```
