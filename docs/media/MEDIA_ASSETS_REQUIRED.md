## Format Recommendations

### Images:
- **Format:** JPG for photos, PNG for graphics with transparency
- **Quality:** 85-90% for JPG (balance between file size and quality)
- **Optimization:** Use tools like ImageOptim, Squoosh, or Sharp
- **WebP:** Consider generating WebP versions for modern browsers (with JPG fallback)

### Videos:
- **Format:** MP4 (H.264) for compatibility
- **Resolution:** 1920 × 1080px minimum
- **Poster frames:** Generate static JPG poster images (1600 × 1200px)

### GIFs:
- **Format:** GIF for animated content
- **Dimensions:** Match image dimensions above
- **Optimization:** Use tools like gifsicle to reduce file size

---

## File Naming Convention

All assets should follow this pattern:
- **Hero images:** `{project-slug}-hero.jpg`
- **Card/thumbnail images:** `{project-slug}-card.jpg`
- **Gallery images:** `{project-slug}-{descriptive-name}.jpg`
- **Video files:** `{project-slug}-{descriptive-name}.mp4`
- **Video posters:** `{project-slug}-{descriptive-name}-poster.jpg`

## Notes

1. **Responsive Images:** Consider generating multiple sizes (1x, 2x) for different screen densities
2. **Lazy Loading:** Gallery images use lazy loading, so prioritize hero/carousel images first
3. **CDN:** All assets are served from `https://media.jpstas.com/`
4. **Caching:** Ensure proper cache headers are set for optimal performance
5. **Accessibility:** All images must have descriptive alt text matching the `alt` attributes in JSON files
